# 2025-11-18 (Day One)

## Normative: In PluralRules, set compactDisplay only if notation is "compact"

`Intl.PluralRules` の内部仕様バグ修正で、`compactDisplay` オプションを `notation` が `"compact"` の場合のみ設定するように仕様を改める Normative PR です。現行仕様では `notation` が `"compact"` でない場合にも `compactDisplay` が内部スロットに設定されてしまう不整合があり、これを修正します。実装への影響は軽微で、ホストの既存挙動と一致させるための整合性修正という位置づけです。会場での反対は無く、Normative PR としてマージすることに合意しました。

## Normative: make re-exporting a namespace object in 2 steps behave like 1 step

`export * as ns from "mod"` のように名前空間オブジェクトを再エクスポートする際、2段階(`import * as ns` してから `export { ns }`)で行った場合と1段階で行った場合の挙動が異なってしまっていた仕様バグの修正です。両者が一致するように仕様の評価順序を改めることで、モジュールのリンク・評価フェーズにおける一貫性を確保します。実装間で実害のある差異が出ていたことが報告され、Normative PR として承認されました。

## Normative: Extend `JSON.parse` and `JSON.stringify` for interaction with the source text of primitive values

`JSON.parse` のソーステキストへのアクセス(`source` reviver 引数)に関連して、プリミティブ値の `JSON.stringify` との相互作用を仕様上明確化する Normative PR です。数値や文字列などのプリミティブをラウンドトリップする際にソーステキストがどう扱われるべきかについて、エッジケースの挙動を仕様に追記します。委員会はこの明確化に合意し、マージされました。
