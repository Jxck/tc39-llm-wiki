## An introduction to the EU CRA & how it matters to all of you

- Slides: [link](https://hosted.akiro.se/2026/Ecma/TC39/Cyber%20Resilience%20Act%20primer.pdf)
- AKI が EU Cyber Resilience Act (CRA) の概説を行った。EU single market で製品を売るには準拠が必須で、software やそれを含む product が対象(純粋な SaaS や website は対象外で NIS2 等の別規制)。risk-based approach が要で、14 の essential cybersecurity requirements と vulnerability handling policy の文書化が中心となる。
- technical documentation には SBOM(Ecma が標準化する CycloneDX が利用可)、component 関係の diagram、risk assessment、support period の根拠などが必要。本体は 2027 年 12 月末施行だが、vulnerability handling と reporting は 2026 年 9 月までに整備が必要。
- vulnerability 対応は severe incident や actively exploited vulnerability で 24 時間以内に ENISA へ early warning、72 時間以内に notification、14/30 日以内に RCA。upstream への fix は machine-readable な形(例: PR)で共有が法的要件となる。
- open source は経済活動を伴わない限り法的義務ゼロで、CRA は個人開発者を狙うものではない(scale が判断軸)。Linux Foundation / Eclipse / OpenJS 等は open source steward に該当しうるが、ECMA-262 は現状 product でも open source project でもないため Ecma に義務はない。spec 自体の脆弱性は product 側の報告責任(JHD は TG3 convener として倫理的責任は負うと補足)。
- 結論(Speaker's Summary): CRA はほぼ全ての product 系企業に影響する。open source は罰則から保護されるが、実務上は cybersecurity 要件への整合が望ましい。SHN の提案で今後の plenary でも 30 分枠のフォローアップを設ける方向。

## agreeing to consider impact of RegExp proposals to linear implementations

- Slides: [link](https://aurele-barriere.github.io/ecma_amsterdam_handout.pdf)
- AUR が CPC・MF・MBH と共に、RegExp proposal が linear-time matching 実装に与える影響を考慮することへの合意を求めた。formal methods/verification の立場から、backreference を持たない JS regex は linear time で matching 可能であることを示した。
- ECMAScript spec は linear-time matching に特に適しており、quantifier には ad-hoc な調整を要するものの、全ての lookaround を linear time で扱える方法が知られているのは ECMAScript だけという点を強調(他の主要言語にない固有の性質)。
- ReDoS 脆弱性の多さを背景に、linearity を user が制御/判断できる facility の検討を動機とした。複数 delegate が catastrophic backtracking のリスクから regex 利用を自制していると報告。
- 結論: 新 proposal の linearity への影響を考慮することに広い支持。今後 regex proposal は regex-matching complexity の impact assessment を含めることで consensus。非 blocking で「complexity の分析が難しい」懸念も出たが、非線形性は自動却下の理由にしないと整理。AUR/CPC/MF が linear matching の preference 指定・linear 実行・linear 可否の判定・time limit など linearity 関連 proposal を持ち帰る。
